The Responsive grid generator

Creating a responsive grid system just became a total breeze...

What is it?

Gridpak is the starting point for your responsive projects, improving your workflow and saving time. Create your responsive grid system once using the simple interface and let Gridpak do the heavy lifting by generating PNGs, CSS and Javascript.

Can’t believe it’s that easy? Watch the demo
Why should I use it?

Put simply – it’s the most flexible system out there. You control every aspect of the grid from column padding to where ‘break points’ occur. Our simple interface makes it quick and easy to visualise your responsive grid system and output it in the formats you need to make a lightning quick start to any project

What's included in the download?

Your Gridpak is a download file that contains:

● PNGs

A PNG version of each grid you have created, ready and waiting for your graphics program of choice.

● Presentational CSS

You’ll have several options for ‘presentational’ CSS.

Gripak.CSS

A straightforward CSS which contains the appropriate media queries and pre-defined selectors based on the grids you have created. Using this file doesn’t require a compiler and will result in a smaller file size but is less semantic than .LESS and .SCSS files.

How to use

Add the selector corresponding to the desired width of your element.

<div class="span_2"></div>

This would create an element two columns wide. The required presentational classes can also be added to existing classes:

<div class="secondary_header span_8"></div>

This would create an element eight columns wide.

If your element appears at the end of a row you will also need to give it the class ‘end’.

<div class="recipe span_8"></div>
    <div class="ingrediants span_2"></div>
    <div class="sidebar span_2 end"></div>

By adding the ‘end’ class to the side bar you are removing any right padding or margin allowing your elements to fit correctly within their container.

Gripak.LESS & Gripak.SCSS

These options also include the appropriate media queries. However, unlike the Gridpak-styles.CSS these files use a combination of ‘Mixins’ and variables rather than selectors, allowing your markup to be more semantic.

How to use

Add the Mixin corresponding to the desired width inside the selector, alongside any other declerations.

.recipe {
    .span(2);
    background: #FFF;	
}.LESS
.recipe {
    @include span(2);
    background: #FFF;	
}.SCSS

These two examples would result in an element that is two columns wide. If your element appears at the end of a row you will also need to include the ‘end’ Mixin.

.recipe {
    .span(2);
    .end();
    background: #FFF;	
}.LESS
.recipe {
@include span(2);
@include end;
background: #FFF;	
}.SCSS

By adding the ‘end’ Mixin to the recipe element you are removing any right padding or margin allowing your elements to fit correctly within their container.

● Browser overlay

A JavaScript file which allows you to toggle your grid on & off in any browser using either the ‘G’ key to toggle the whole grid, the ‘V’ key to toggle the vertical grid or the ‘B’ key to toggle the baseline. Simply add the following line to the head of your document and you’re on your way.

<script type="text/javascript" src="gridpak.js"></script>

Please note:

The JavaScript file appends the grid directly after your body tag. It does this because it assumes that this is the main content container. If this is not the case and your main content container is called something like ‘#content_wrapper’ then please change the following variable in the JavaScript file.
