The Responsive grid generator

Creating a responsive grid system just became a total breeze...

What is it?

Gridpak is the starting point for your responsive projects, improving your workflow and saving time. Create your responsive grid system once using the simple interface and let Gridpak do the heavy lifting of generating CSS, PNGs and Javascript.

Don't believe us? Watch the demo
Why should I use it?

It’s the most flexible system out there – you control every aspect of the grid from column padding to where ‘break points’ occur. Our simple interface makes it quick and easy to visualise your responsive grid system and output it in the formats you need.

What's included in the download?

● PNGs

A PNG version of each grid you have created, ready and waiting for your graphics program of choice.

● Presentational CSS

Your download provides you with three options when it comes to ‘presentational’ CSS.

Gripak.CSS

The first is a straight forward CSS which includes the appropriate media queries and pre-defined selectors based on the grids you have created. Whilst using this file doesn’t require a compiler and will result in a smaller file size it is less semantic than the .LESS and .SCSS files.

How to use

Add the selector corresponding to the desired width of your element followed by the class ‘column.

<div class="two column"></div>

This would create an element two columns wide. The required presentational classes can also be added to existing classes:

<div class="secondary_header eight column"></div>

This would create an element eight columns wide.

If your element appears at the end of a row you will also need to give it the class ‘end’.

<div class="recipe eight column"></div>
<div class="ingrediants two column"></div>
<div class="sidebar two column end"></div>

By adding the ‘end’ class to the side bar you are removing any right padding or margin allowing your elements to fit correctly within their container.

Gripak.LESS & Gripak.SCSS

The second and third options are a .LESS and a .SCSS file which also include the appropriate media queries. However, unlike the Gridpak-styles.CSS these files use a combination of ‘Mixins’ and variables not selectors, allowing your markup to be more semantic.

How to use

Add the mixin corresponding to the desired width inside the selector, alongside any other declerations.

.recipe {
    .two();
    background: #FFF;	
} 
.recipe {
    @include two;
    background: #FFF;	
}

These two examples would result in an element that is two columns wide.

If your element appears at the end of a row you will also need to include the ‘end’ mixin.

.recipe {
    .two();
    .end();
    background: #FFF;	
} 
.recipe {
    @include two;
    @include end;
    background: #FFF;	
}

By adding the ‘end’ mixin to the recipe elemtent you are removing any right padding or margin allowing your elements to fit correctly within their container.

● Browser overlay

A JavaScript file which allows you to toggle your grid on & off in any browser using either: the ‘G’ key to toggle the whole grid, the ‘V’ key to toggle the vertical grid or the ‘B’ key to toggle the baseline.

Simply add the following line to the head of your document and you’re away.

<script type="text/javascript" src="gridpak.js"></script>
Please note:

The JavaScript file ‘appends the grid directly after your body tag. It does this because it assumes that this is the main content container.
